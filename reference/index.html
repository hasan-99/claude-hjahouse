Slash Commands

/help

Show available commands and usage hints.

/clear

Start a fresh conversation. CLAUDE.md instructions stay active.

/compact

Summarize conversation to reduce context usage. Accepts focus instructions.

/compact focus on the auth module
/context

Show context window usage with a visual breakdown of token allocation.

/diff

Open an interactive viewer for uncommitted changes.

/model

Switch between Sonnet, Opus, and Haiku mid-session.

/model opus
/cost

Show session token usage and estimated cost.

/status

Show current version, model, and account info.

/cd

Move the session to a new working directory without breaking the prompt cache mid-session.

/cd ../api
/doctor

Run a health check on your Claude Code installation.

/init

Scan your project and generate a starter CLAUDE.md.

/memory

View and edit CLAUDE.md memory files (global, project, auto).

/review

Review code changes on the current branch with suggestions.

/permissions

View and manage tool permissions. Configure in .claude/settings.json.

/config

Open Claude Code configuration and settings.

/login

Switch Anthropic accounts.

/branch

Fork conversation into a parallel branch to explore alternatives.

/rewind

Roll back to a previous message and undo file changes after that point.

/resume

Resume a previously saved session by name or ID.

/resume auth-refactor
/rename

Rename the current session for easier recall later.

/export

Export the conversation to a markdown file.

/export session-review.md
/effort

Set reasoning depth: low, medium, high, xhigh, or max (session-only). /effort ultracode adds automatic workflow orchestration on top of xhigh.

/effort high
/plan

Enter planning mode — Claude researches first, then presents a plan for approval.

/plan migrate from REST to GraphQL
/btw

Ask a side question without adding it to conversation history.

/btw what's the syntax for a TS generic constraint?
/batch

Split work across parallel agents in isolated git worktrees.

/batch add JSDoc to all exported functions
/loop

Run a task on a recurring interval within your session.

/loop 5m check if the build succeeded
/schedule

Create a cloud-backed scheduled task that runs even when offline.

/schedule "run security audit every Monday at 9am"
/workflows

List and manage running and completed dynamic workflows; open a run's progress view, and press s to save a run's script as a /command. Start a workflow by including 'ultracode' in your prompt or with /effort ultracode.

/deep-research

Bundled dynamic workflow that fans out web searches, cross-checks sources, and returns a cited report with weak claims filtered out. Requires the WebSearch tool.

/deep-research what changed in the Node.js permission model between v20 and v22?
/debug

Toggle verbose mode to see tool calls and thinking steps. Also Ctrl+O.

/code-review

Review the current diff for correctness bugs. Accepts an effort level (low/medium/high/xhigh/max/ultra) and --comment to post inline comments on the current GitHub PR.

/code-review high --comment
/code-review ultra

Run a deep, multi-agent code review in a cloud sandbox using parallel analysis and critique. /ultrareview is an alias.

/code-review ultra 128
/simplify

Cleanup-only review that applies fixes (reuse, simplification, efficiency, abstraction level) without hunting for bugs.

/security-review

Security-focused review of pending changes. Read-only — never edits files.

/agents

List, create, edit, or remove subagent definitions.

/mcp

Show active MCP server connections and available tools.

/plugin

Manage plugins — install, list, remove, or reload.

/plugin install pr-review
/reload-plugins

Hot-reload all plugin files during development.

/goal

Set a completion condition Claude works toward across turns. Shows live elapsed time, turns, and tokens.

/goal migrate all API endpoints from REST to GraphQL
/recap

Generate a one-line summary of the current session. Also runs automatically when returning to the terminal after stepping away.

/fewer-permission-prompts

Scan recent transcripts for common read-only tool calls and propose an allowlist to reduce permission prompts.

/tui

Switch between classic and flicker-free fullscreen rendering mid-conversation.

/focus

Toggle focus view for distraction-free input.

/sandbox

Enable OS-level isolation for file system and network access.

/usage-credits

View and manage usage credits on your account. Renamed from /extra-usage in v2.1.144; the old name still works.

CLI

CLI Flags

claude -p "prompt"

Run a one-shot prompt non-interactively. Foundation for CI/CD integration.

echo "$DIFF" | claude -p "review these changes for security issues"
--output-format json

Return structured JSON output. Useful for parsing in scripts and pipelines.

claude -p "analyze this" --output-format json
--model <name>

Override the default model for a single invocation.

claude --model opus "redesign the database schema"
--permission-mode <mode>

Start in a specific permission mode (default, acceptEdits, plan, auto, dontAsk, or bypassPermissions).

claude -p "run tests" --permission-mode bypassPermissions
--sandbox

Enable OS-level isolation for safe automated analysis.

--safe-mode

Start with all customizations (CLAUDE.md, plugins, skills, hooks, MCP servers) disabled for troubleshooting. Same as CLAUDE_CODE_SAFE_MODE=1.

--max-turns <n>

Cap execution to n turns. Useful for time-limiting automated runs.

--no-session-persistence

Don't save session data. Good for disposable automation tasks.

--resume

Resume the most recent Claude Code session.

--continue

Continue a paused workflow from the current repository.

--agent <name>

Start a session with a specific subagent.

claude --agent security-reviewer
--plugin-dir <path>

Load a plugin for this session only (for testing).

claude --plugin-dir ./my-plugin
--bare

Cleanest output for scripted usage. No formatting or decoration.

--worktree

Run in an isolated git worktree for experimental work.

--dangerously-skip-permissions

Skip all permission prompts. Equivalent to --permission-mode bypassPermissions.

claude project purge [path]

Delete all Claude Code state for a project. Supports --dry-run, --all, -y, and -i flags.

claude project purge --dry-run
--plugin-url <url>

Fetch a plugin .zip from a URL for the current session. Repeatable for multiple plugins.

claude --plugin-url https://example.com/my-plugin.zip
claude agents

Open the agent view — a roster of all sessions showing state, last activity, and running status.

claude agents --json

Print the session roster as a JSON array for scripting.

claude agents --json | jq '.[] | select(.status == "waiting")'
claude plugin details <name>

Show a plugin's component inventory and projected token cost per session.

claude plugin details pr-review
claude plugin prune

Remove auto-installed plugin dependencies no longer required by any installed plugin.

claude plugin uninstall pr-review --prune
claude plugin enable <name>

Re-enable a previously disabled plugin without re-installing. Accepts --scope user|project|local.

claude plugin enable formatter --scope project
claude plugin disable <name>

Disable an installed plugin without uninstalling it.

claude plugin disable formatter --scope project
--add-dir <path>

Extend working directories with read/edit access for this session. Repeatable.

claude --add-dir ../shared-types
--mcp-config <file>

Load MCP servers from one or more JSON files. Pair with --strict-mcp-config to ignore other sources.

claude --strict-mcp-config --mcp-config ./repro.json
--channels <plugin>

Enable channel plugins that push events (chat messages, webhooks) into the running session.

claude --channels plugin:telegram@claude-plugins-official
claude auth login

Authenticate with OAuth. In headless environments (WSL2, SSH), paste the code manually.

Config

Configuration Files

CLAUDE.md

Project-level instructions, conventions, and workflow notes. Committed to git and shared with the team.

CLAUDE.local.md

Personal overrides for CLAUDE.md. Git-ignored, not shared.

.claude/settings.json

Project settings: permissions, hooks, MCP servers. Committed to git.

{ "permissions": { "allow": ["Bash(npm run test)"], "deny": ["Bash(rm -rf /)"] } }
.claude/settings.local.json

Personal project settings. Git-ignored, overrides .claude/settings.json.

~/.claude/CLAUDE.md

Global user instructions that apply to all projects.

~/.claude/settings.json

Global user settings that apply to all projects.

.claude/skills/

Project-scoped custom skills (SKILL.md files). Committed to git.

.claude/agents/

Project-scoped subagent definitions. Committed to git.

.claude/rules/*.md

Path-scoped rules. Use frontmatter paths: field to target specific files.

--- paths: src/api/**/*.ts --- Always validate input parameters.
.mcp.json

Project MCP server configuration. Committed to git, shared with team.

~/.claude.json

User/local MCP server configuration.

MCP

MCP Setup

claude mcp add <name> <uri>

Register a new MCP server. Supports HTTP and stdio transports.

claude mcp add --transport stdio github -- npx @modelcontextprotocol/server-github
claude mcp add --header

Add an MCP server with authentication headers.

claude mcp add --header "Authorization: Bearer $TOKEN" api https://api.example.com/mcp
claude mcp list

List all configured MCP servers with transport and connection status.

claude mcp get <name>

Show details for a specific MCP server.

claude mcp remove <name>

Remove an MCP server configuration.

claude mcp add-from-claude-desktop

Import MCP server configurations from Claude Desktop.

/mcp

Show active connections in-session and trigger OAuth flows.

mcp__server__tool

MCP tools appear namespaced. Use naturally in conversation.

Use the GitHub MCP to list open PRs older than 3 days.
Hooks

Hooks

PreToolUse

Runs before a tool executes. Can block the action (exit code 2).

Validate Bash commands before execution, block dangerous patterns.
PostToolUse

Runs after a tool completes. Use for formatting, linting, or logging.

Auto-format files after Edit/Write with prettier.
UserPromptSubmit

Intercept user input before Claude processes it.

Stop

Runs when Claude finishes responding. Check completion criteria.

Verify all tests pass before marking a task complete.
SubagentStart / SubagentStop

Track subagent lifecycle for orchestration and logging.

Hook types

command (shell), prompt (LLM evaluation), agent (subagent), http (webhook).

Hook matchers

Filter which tools trigger hooks: exact name, regex, or * for all.

matcher: "Edit|Write" — only trigger on file modifications.
Hook args (exec form)

Set args as an array to spawn without a shell. Omit for shell tokenization.

"command": "node", "args": ["--check", "file.js"]
Skill-level hooks

Define hooks in SKILL.md frontmatter. Scoped to that skill only.

Stop / SubagentStop input fields

Hook input includes background_tasks (running bash/subagents) and session_crons (queued scheduled tasks).

if data['background_tasks'] or data['session_crons']: print('{"decision":"block"}')
Access

Permissions

default

Ask before write/edit/bash operations. Read, Glob, Grep always allowed.

plan

Research and present plans only. No file modifications until approved.

acceptEdits

Allow file edits without prompting. Still ask for Bash commands.

auto

Allow all operations without prompting. Use in trusted environments.

bypassPermissions

Skip all safety checks. Only for fully automated CI/CD pipelines.

Allow patterns

Pre-approve specific tools in .claude/settings.json.

"allow": ["Bash(npm run test)", "Bash(git *)", "Read", "Edit"]
Deny patterns

Block dangerous operations regardless of permission mode.

"deny": ["Bash(git push --force*)", "Bash(rm -rf /)"]
Agent

Subagents

@"agent-name"

Invoke a specific agent inline during conversation.

@"security-reviewer" audit the auth module
claude --agent <name>

Start a full session with a specific agent from the CLI.

.claude/agents/*.md

Define project-scoped agents with frontmatter for tools, model, effort.

Built-in agents

general-purpose, Explore (Haiku, read-only), Plan (research first).

background: true

Run agent in background. Use Ctrl+B to background a running agent.

Env

Environment Variables

CLAUDE_EFFORT

Current effort level (low/medium/high/xhigh/max). Available in Bash subprocesses and hooks.

CLAUDE_CODE_SESSION_ID

Unique session identifier. Available in Bash subprocesses and hooks for tracking and correlation.

CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN

Set to 1 to keep conversation in terminal scrollback instead of fullscreen alternate screen.

CLAUDE_CODE_FORCE_SYNC_OUTPUT

Set to 1 to force-enable synchronized output rendering on terminals where auto-detection fails.

CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE

Set to 1 to enable automatic background updates for Homebrew/WinGet installations.

CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY

Set to 1 to populate the /model picker from your LLM gateway's /v1/models endpoint.

CLAUDE_CODE_HIDE_CWD

Set to 1 to hide the working directory from the startup banner.

CLAUDE_CODE_SAFE_MODE

Set to 1 (or pass --safe-mode) to start with all customizations — CLAUDE.md, plugins, skills, hooks, MCP servers — disabled for troubleshooting.

CLAUDE_CODE_DISABLE_BUNDLED_SKILLS

Set to 1 to hide bundled skills, workflows, and built-in slash commands from the model.

DISABLE_UPDATES

Set to 1 to block all update paths including manual claude update.

CLAUDE_CODE_USE_POWERSHELL_TOOL

Set to 1 to enable the PowerShell tool on Linux/macOS/WSL (requires pwsh 7+). On Windows set to 0 to opt out of the rollout.

CLAUDE_CODE_DISABLE_WORKFLOWS

Set to 1 to turn off dynamic workflows. Read at startup. You can also toggle them in /config or set disableWorkflows in settings.

Config

Config Options

worktree.baseRef

Controls worktree branching: 'fresh' (default, from remote) or 'head' (from local HEAD).

sandbox.bwrapPath / sandbox.socatPath

Managed-only (Linux/WSL2). Absolute paths to bubblewrap and socat binaries for sandbox.

parentSettingsBehavior

Admin-tier. Controls SDK/IDE parent settings merge: 'first-wins' (default) or 'merge'.

cleanupPeriodDays

Days before session files and orphaned worktrees are auto-deleted (default: 30).

prUrlTemplate

Custom URL template for the footer PR badge. Supports {owner}, {repo}, {number} placeholders.

disableBundledSkills

Set to true to hide bundled skills, workflows, and built-in slash commands from the model. CLAUDE_CODE_DISABLE_BUNDLED_SKILLS is the env-var equivalent.

enforceAvailableModels

Managed setting (v2.1.175). When enabled, the availableModels allowlist also constrains the Default model, not just alternate models.

footerLinksRegexes

Settings (v2.1.176) for regex-matched link badges in the footer row, configurable via user or managed settings.

Flow

Common Workflows

/effort high → /plan → approve → implement

Deep work: set high reasoning, plan first, then execute.

/diff → /cost → /export → /compact

End-of-session: review changes, check cost, export, then compact.

/batch <instruction>

Large refactors: split work across parallel agents in isolated worktrees.

/loop 5m <check>

Monitoring: poll build status, error logs, or deploy health on interval.

/branch → experiment → /resume

Exploration: branch conversation, try an approach, resume if it fails.

echo $DIFF | claude -p "review" --output-format json

CI/CD: pipe diffs into Claude for automated code review with JSON output.

/init → edit CLAUDE.md → commit

Project setup: generate instructions, customize, share with team.
