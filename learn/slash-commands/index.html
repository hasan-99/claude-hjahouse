Slash Commands

Slash commands are the fastest way to control Claude Code’s behavior during an interactive session. Type /at any prompt to see the full list, or type a few letters to filter. This module covers the built-in commands you’ll use every day.

Discovering Commands

Type / at the prompt and a menu appears with all available commands. Start typing to filter — /co narrows to /compact, /color, /config, /context, /cost, /copy. Arrow keys navigate, Enter selects. Commands that aren’t available for your current setup are hidden automatically, so you only see what works.

slash command menu
>
✦New to Claude Code?
Try /powerup — it runs interactive lessons with animated demos that walk you through key features right inside the CLI.

Some commands accept arguments directly: /compact focus on the API layer, /model opus, /effort high, /rename auth-refactor. Others like /context, /cost, and /status run immediately with no arguments.

example
/compact focus on the payment service

Command Categories

Built-in commands group into a few categories. Knowing the categories helps you find the right command without memorizing all of them.

Context Management

Controls how much of the conversation Claude can see.

/context

Shows a visual grid of your context usage.

/compact [instruction]

Compresses the conversation. Pass instructions to control what's preserved.

/clear

Starts completely fresh — wipes the conversation history.

context management
/context
/compact keep the auth refactor plan, drop the debugging
/clear

Session Tools

Manage and revisit work across sessions.

/rename my-feature

Gives the session a readable name for easy resumption.

/resume

Picks up a previous named session.

/branch

Creates a parallel conversation to explore an alternative without losing your current state.

/rewind (alias: /undo)

Rolls back to an earlier point in the conversation.

/export

Saves the session to a file or clipboard.

/recap

Generates a one-line summary of what happened. Also runs automatically when you return to the terminal.

Configuration

Adjust Claude’s behavior mid-session without leaving the terminal.

/model

Switches between available models — Sonnet, Opus, Haiku, and aliases like best or opusplan.

/effort low|medium|high|xhigh|max|auto

Sets reasoning depth for the current session.

/permissions

Manages what Claude can do without asking for approval.

/config

Opens the settings menu.

/theme

(v2.1.118+) Creates and switches between named custom themes stored as JSON in ~/.claude/themes/.

/tui

Switches between classic and flicker-free fullscreen rendering mid-conversation.

/focus

Toggles focus view for distraction-free input.

/fewer-permission-prompts

Scans recent transcripts and proposes an allowlist for .claude/settings.json to reduce future prompts.

Diagnostics

Help when something isn’t working.

/cost

Shows session cost, duration, code changes, and token usage.

/usage-credits

(v2.1.144+, formerly /extra-usage) View and manage usage credits on your account.

/status

Shows version, model, and account info.

/doctor

Checks installation health.

/diff

Opens an interactive viewer for uncommitted changes — useful for reviewing what Claude has done before committing.

Pre-ship Reviews

Check your work before it leaves your branch.

/code-review [effort] [--comment] [--fix]

Reviews the current diff for correctness bugs. Accepts an effort level (low/high), --comment to post findings as inline PR comments, and --fix to apply findings to your working tree.

/code-review ultra [PR#]

Deep multi-agent review in a cloud sandbox using parallel analysis. With no args reviews the current branch; pass a PR number to review a GitHub PR. /ultrareview is an alias.

/review

Read-only deeper pass on the change set.

/security-review

Security-focused review of pending changes.

ⓘArgument patterns
Commands that accept free-text arguments (/compact, /rename) take everything after the command name as the argument — no quotes needed. Commands that accept flags (/code-review --fix, /code-review high) follow standard CLI conventions.

Custom Slash Commands

Beyond the built-ins, you can define your own slash commands as skills. A skill is a Markdown file placed in ~/.claude/skills/ (user-global) or .claude/skills/ (project-local). Claude discovers the file and exposes it as /filename in the menu.

~/.claude/skills/deploy.md
# deploy

Run pre-flight checks and deploy the current branch to staging.

## Steps
1. Run `npm test` — abort if tests fail
2. Run `npm run build` — abort on build errors
3. Run `vercel --prod` and print the deployment URL
4. Ping #deploys in Slack with the URL

After saving that file, type /deploy at any Claude Code prompt and the skill runs automatically. Skills can include acceptance criteria, context references, and structured action steps — see the Skills module for the full format.

▲Project vs. user skills
Skills in .claude/skills/ (project-local) override same-named skills in ~/.claude/skills/ (user-global). Keep generic commands at the user level and project-specific ones in the repo so teammates share them via version control.
