أوامر السلاش

أوامر الـ slash commands هي أسرع طريقة تتحكم بيها في سلوك Claude Code خلال جلسة تفاعلية. اكتب / في أي وقت وهتظهر لك قائمة بكل الأوامر المتاحة، أو اكتب أول كام حرف عشان تفلتر النتايج. الوحدة دي بتغطي الأوامر المدمجة اللي هتستخدمها كل يوم.

اكتشاف الأوامر

اكتب / في الـ prompt وهتظهر قائمة بكل الأوامر المتاحة. ابدأ تكتب عشان تفلتر — /co هتوصّلك لـ /compact، /color، /config، /context، /cost، /copy. استخدم الأسهم للتنقل، وـ Enter للاختيار. الأوامر اللي مش متاحة في الإعداد الحالي بتاعك بتتخفى تلقائيًا، يعني هتشوف بس اللي شغّال معاك.

قائمة أوامر الـ slash
>
✦جديد على Claude Code؟
جرّب /powerup — بيشغّل دروس تفاعلية مع animated demos بتعرّفك على الميزات الأساسية جوه الـ CLI مباشرة.

بعض الأوامر بتقبل arguments مباشرة: /compact focus on the API layer، /model opus، /effort high، /rename auth-refactor. وأوامر تانية زي /context، /cost، و/status بتشتغل فورًا من غير arguments.

مثال
/compact focus on the payment service

تصنيفات الأوامر

الأوامر المدمجة بتتقسّم لكام مجموعة. لما تعرف التصنيفات، هتلاقي الأمر الصح بسرعة من غير ما تحفظ كل حاجة.

إدارة الـ Context

بتتحكم في قد إيه Claude شايف من المحادثة.

/context

بيعرض grid بصري بيوضّح استهلاك الـ context بتاعك.

/compact [تعليمة]

بيضغط المحادثة. ممكن تمرّر تعليمات تحدد إيه يتحفظ: /compact keep the migration plan, drop the debugging

/clear

بيبدأ جلسة جديدة تمامًا — بيمسح تاريخ المحادثة كله.

إدارة الـ context
/context
/compact keep the auth refactor plan, drop the debugging
/clear

أدوات الجلسة

بتخلّيك تدير شغلك وترجع له عبر الجلسات.

/rename my-feature

بيدي الجلسة اسم واضح للرجوع إليها بسهولة.

/resume

بيرجعك لجلسة قديمة مُسمّاة.

/branch

بيعمل محادثة موازية تقدر تستكشف فيها بديل من غير ما تفقد الحالة الحالية.

/rewind (alias: /undo)

بيرجعك لنقطة أبكر في المحادثة.

/export

بيحفظ الجلسة في ملف أو الـ clipboard.

/recap

بيولّد ملخص من سطر واحد عن اللي حصل. كمان بيشتغل تلقائيًا لما ترجع للتيرمنال.

أوامر الإعدادات

بتغيّر سلوك Claude في نص الجلسة من غير ما تخرج من التيرمنال.

/model

بيتنقل بين الموديلات المتاحة زي Sonnet و Opus و Haiku، و aliases تانية زي best أو opusplan.

/effort low|medium|high|xhigh|max|auto

بيحدد عمق التفكير للجلسة الحالية.

/permissions

بيتحكّم في إيه Claude يقدر يعمله من غير ما يسألك.

/config

بيفتح قائمة الإعدادات.

/theme (جديد في v2.1.118)

بيعمّل وبيتنقل بين custom themes مُسمّاة. الـ themes بتتخزّن كملفات JSON في ~/.claude/themes/.

/tui

بيتنقل بين الـ rendering الكلاسيكي والـ fullscreen الخالي من الـ flicker وسط المحادثة.

/focus

بيفعّل ويقفل وضع التركيز للإدخال بدون تشتيت.

/fewer-permission-prompts

بيمسح الـ transcripts الأخيرة ويقترح allowlist لـ .claude/settings.json عشان تقلل permission prompts.

التشخيصات

بتساعدك لما حاجة مش شغّالة.

/cost

بيوضّح تكلفة الجلسة، المدة، تغييرات الكود، واستهلاك الـ tokens.

/usage-credits (جديد في v2.1.144)

اعرض وأدِر الـ usage credits على حسابك. الاسم القديم /extra-usage لسه شغّال.

/status

بيعرض الـ version، الـ model، ومعلومات الحساب.

/doctor

بيعمل فحص على سلامة التثبيت.

/diff

بيفتح عارض تفاعلي للتغييرات اللي لسه ما اتعملش لها commit — مفيد لما تبص على اللي Claude عمله قبل الـ commit.

مراجعات ما قبل الـ Ship

افحص شغلك قبل ما يخرج من الـ branch بتاعك.

/code-review [effort] [--comment] [--fix]

بيراجع الـ diff الحالي عشان يلاقي correctness bugs. بيقبل effort level (/code-review high للتغطية الأوسع) و--comment لنشر النتايج كـ inline PR comments و--fix لتطبيق الإصلاحات.

/code-review ultra [رقم PR]

مراجعة عميقة متعددة الـ agents في الـ cloud بتحليل متوازٍ. من غير arguments بيراجع الـ branch الحالي؛ مرّر رقم PR لمراجعة GitHub PR. /ultrareview هو alias.

/review

قراءة معمّقة read-only للتغييرات.

/security-review

مراجعة بتركيز أمني للتغييرات المعلّقة.

الأوامر في سير عمل حقيقي

إليك تسلسل نموذجي: ضغط الـ context لتحرير المساحة، الانتقال لموديل أقوى لمهمة صعبة، ثم مراجعة التكلفة قبل الانتهاء.

ديمو سير العمل
>
تسلسل أوامر شائع
/context
/compact keep the auth refactor plan
/model opus
/effort high
/cost
/code-review high
/code-review --comment
ⓘأنماط الـ Arguments
الأوامر اللي بتقبل نص حر (/compact، /rename) بتاخد كل حاجة بعد اسم الأمر كـ argument — من غير ما تحتاج علامات تنصيص. الأوامر اللي بتقبل flags (/code-review --fix، /code-review high) بتتبع اتفاقيات الـ CLI العادية.

أوامر Slash مخصصة

بعيدًا عن الأوامر المدمجة، تقدر تعرّف أوامر slash خاصة بيك عن طريق Skills. الـ skill هو ملف Markdown بتحطّه في ~/.claude/skills/ (للمستخدم عمومًا) أو .claude/skills/ (للمشروع). Claude بيكتشف الملف ويعرضه كـ /filename في القائمة.

~/.claude/skills/deploy.md
# deploy

شغّل الـ pre-flight checks وانشر الـ branch الحالي على staging.

## Steps
1. شغّل `npm test` — وقف لو الـ tests فشلت
2. شغّل `npm run build` — وقف لو في أخطاء build
3. شغّل `vercel --prod` وأطبع رابط الـ deployment
4. أرسل رابط الـ deployment في قناة #deploys على Slack

بعد حفظ الملف ده، اكتب /deploy في أي prompt في Claude Code والـ skill هيشتغل تلقائيًا. الـ Skills تقدر تتضمن acceptance criteria ومراجعة الـ context وخطوات action مفصّلة — شوف وحدة المهارات عشان تعرف الصيغة الكاملة.

▲Skills المشروع مقابل skills المستخدم
الـ Skills في .claude/skills/ (على مستوى المشروع) بتتغلّب على الـ skills بنفس الاسم في ~/.claude/skills/ (على مستوى المستخدم). حافظ على الأوامر العامة على مستوى المستخدم والأوامر الخاصة بالمشروع في الـ repo عشان الفريق يشاركها عبر الـ version control.

تحقّق من فهمك

1. إيه اللي بيحصل لما تكتب / في الـ prompt بتاع Claude Code؟

2. أنهي أمر بيضغط المحادثة مع إمكانية الاحتفاظ بأجزاء معينة؟

3. إزاي تعرف قد إيه من نافذة الـ context بتاعتك اتاستخدم؟

4. إيه اللي بيعمله /effort high؟
