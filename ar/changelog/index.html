التفرقة بين timeout بدء تشغيل الـ MCP وtimeout تنفيذ الأدوات (يتجاوز audit PR #312)

كان الموديول يغطّي timeout بدء التشغيل فقط ولا يذكر timeout تنفيذ الأدوات المنفصل. هذا التمييز موجود الآن في موديول الـ mcp، إنجليزي وعربي.

تم التحسينmcp: التفرقة بين timeout بدء تشغيل الـ server وtimeout تنفيذ الأدوات على مستوى كل server

MCP_TIMEOUT يحدد timeout بدء تشغيل/اتصال الـ server عند الإطلاق. تنفيذ الأدوات له حد منفصل: حقل timeout بالميلي ثانية في إدخال الـ server في .mcp.json (مثلًا "timeout": 600000 لعشر دقائق) يحدد أقصى مدة لأي tool call واحد، ويتجاوز MCP_TOOL_TIMEOUT لهذا الـ server فقط، وهو حد زمني صارم لا يمتد بإشعارات التقدّم، ويتجاهل القيم الأقل من 1000.

تم الإصلاحmcp: رفض إعادة صياغة MCP_TIMEOUT غير الدقيقة من audit PR #312

أراد audit PR #312 إعادة تسمية MCP_TIMEOUT كـ 'timeout بدء تشغيل خاص بكل server، لا timeout عام'. الـ docs تسمّيه 'MCP server startup timeout' — متغير بيئة وقت بدء التشغيل، لا إعدادًا على مستوى كل server — فالجملة المقترحة الركيكة حُذفت لصالح التمييز الدقيق بين timeout بدء التشغيل وtimeout تنفيذ الأدوات.

العربيةتماثل عربي لتمييز timeout الـ MCP

موديول الـ mcp العربي يعكس التمييز بين timeout بدء التشغيل (MCP_TIMEOUT) وtimeout تنفيذ الأدوات (حقل timeout على مستوى كل server / MCP_TOOL_TIMEOUT)، مطابقًا للتغيير الإنجليزي.
